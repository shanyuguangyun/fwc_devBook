# 权限设计

> 1.思考
```shell
因为想设计的容易理解。
所以一开始想的就是RBAC模型。

但是RBAC模型所有的权限都是基于分配给角色再计算出用户权限的，
而且我希望的一个问题是系统后续维护的时候，如果新增功能菜单，不需要手动更新角色权限。
还一个希望是，不希望配置的角色过多，最后角色即便冗余了，但是因为已经配置给用户使用了，也不敢进行删除。无法进行管理。

构思设计三种用户授权模式，1、基于角色。2、直接授权。3、混合模式（角色+授权）
构思是否采取RBAC的变形，如增加用户组、增加继承功能。
构思是否采取部门角色，如此新增用户的时候若给他设置了部门，则默认有部门角色权限。
```
